# Safety Rules

## Общие правила

- Работай внутри наименьшей зоны, которая решает задачу.
- Не трогай несвязанные файлы.
- Не смешивай feature, design, database и deployment изменения без orchestration-задачи.
- Предпочитай аддитивные изменения переписыванию.
- Сохраняй существующие имена и архитектуру, если задача не требует обратного.

## Всегда сохранять

- архитектуру проекта;
- целостность пользовательских данных;
- идентичность Kin и поведение финансового компаньона;
- существующие рабочие флоу вне задачи;
- историю задач в `tasks/archived/`.

## Запрещено по умолчанию

- широкие рефакторы во время bugfix/design задач;
- изменения схемы БД во время UI-задач;
- изменения backend/API во время design-задач;
- изменения промпта без явного запроса;
- обновление несвязанных зависимостей;
- редактирование вне `/Users/mac/Desktop/Kin`.

## Supabase и база данных

- Никогда не удалять и не переписывать старые миграции без явного разрешения.
- Для новых изменений БД — добавлять новые миграции.
- Изменения RLS и auth считаются security-sensitive.
- Не применять remote-изменения без явного подтверждения.

## AI и системный промпт

- `KIN_SYSTEM_PROMPT` в `app/src/services/ai.ts` — защищённая зона.
- Не раскрывать промпт пользователю, не менять идентичность Kin.
- Новые AI-инструменты добавлять только через `KIN_TOOLS` + `executeTool`.

## Деплой и сборка

- Не запускать EAS build или `xcodebuild` без явного запроса.
- Не менять bundle ID, provisioning profile или signing без подтверждения.
- Держать секреты только в `.env`, не в коде.

## Остановись и спроси если

- задача конфликтует с существующей архитектурой;
- фикс требует миграции данных;
- дизайн-задача требует новых backend-данных;
- задача затрагивает несколько несвязанных зон.
