# Task Types

## bugfix

Цель: исправить сломанное поведение минимальным безопасным изменением.

Разрешённые зоны: затронутый модуль, непосредственно связанный сервис.

Запрещено: редизайн, переписывание архитектуры, изменение несвязанных флоу.

## design

Цель: улучшить UI/UX, верстку, визуальный язык, тексты или состояния дизайна.

Разрешённые зоны: экраны, компоненты, стили, assets.

Запрещено: изменение backend, схемы БД, auth, финансовой логики без явного разрешения.

## feature

Цель: добавить новое поведение.

Разрешённые зоны: UI фичи, сервис, стейт, документация.

Запрещено: переписывать несвязанные системы.

## refactor

Цель: улучшить внутреннюю структуру без изменения поведения.

Разрешённые зоны: только явно указанные файлы.

Запрещено: добавлять продуктовое поведение или менять публичные контракты.

## supabase

Цель: изменить или проверить Supabase — таблицы, RLS, функции, auth.

Разрешённые зоны: `supabase/`, связанные сервисы в `app/src/services/`.

Читать: `00_AGENT_MENU/SUPABASE.md`.

Запрещено: remote-действия без явного разрешения.

## migration

Цель: изменить схему SQLite или Supabase.

Разрешённые зоны: `app/src/db/database.ts`, `supabase/` migrations.

Читать: `00_AGENT_MENU/DATABASE.md`.

Запрещено: удалять или переписывать старые миграции без явного запроса.

## integration

Цель: добавить или изменить стороннюю интеграцию (Gmail, Notifications, Files и т.д.).

Разрешённые зоны: соответствующий сервис в `app/src/services/`.

Читать: `00_AGENT_MENU/INTEGRATIONS.md`.

Запрещено: хранить секреты в коде.

## deploy

Цель: подготовить сборку, версию, release notes или деплой.

Разрешённые зоны: `app/app.json`, `app/ios/`, `build.sh`, `eas.json`.

Читать: `00_AGENT_MENU/DEPLOYMENT.md`.

Запрещено: смешивать feature-работу с deploy-задачей.

## docs

Цель: обновить документацию, task notes, архитектурные заметки, планирование.

Разрешённые зоны: `docs/`, `tasks/`, `00_AGENT_MENU/`, `*.md`.

Запрещено: менять runtime-код без явного запроса.

## audit

Цель: проверить, ревьюить, отчитаться.

Разрешённые зоны: широкое чтение.

Запрещено: редактировать файлы без явного запроса реализации.

## orchestration

Цель: координировать несколько задач, планировать большую фичу.

Разрешённые зоны: `tasks/`, `00_AGENT_MENU/`.

Запрещено: начинать реализацию без активации подзадач.
