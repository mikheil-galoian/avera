## Short Version

Subject:

`Evidence-first review for simulation, requirements, and verification artifacts`

Body:

`Hi — we built a local workflow called AVERA that turns engineering artifacts like simulation CSV, requirements exports, and verification logs into one stable review path.`

`The point is not a new dashboard. The point is that the same evidence-first kernel can normalize different artifact families, produce traceability, support a release decision, and generate a portable handoff pack.`

`If this is relevant for your team, the best next step is one narrow pilot around one artifact family you already use.`

## Slightly More Technical Version

Subject:

`Local adapted-evidence review path for automotive engineering teams`

Body:

`Hi — we’ve built a local evidence-first review layer called AVERA. It compares baseline vs current verification evidence, links the result to requirements and affected components, and outputs a conservative decision + handoff package.`

`What is new for us now is that AVERA can adapt external artifact families before review, including simulation CSV, richer requirements exports, verification log CSV, and JUnit/xUnit XML.`

`If useful, we’d like to show a 10-minute walkthrough and discuss one narrow pilot around one artifact family your team already has.`
