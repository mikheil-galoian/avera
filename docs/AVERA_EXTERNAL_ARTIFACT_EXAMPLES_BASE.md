# AVERA External Artifact Examples Base

**Date:** 2 May 2026  
**Status:** Preparation layer for real external artifacts  
**Purpose:** Define what real external artifact examples AVERA needs next and how to collect them

## Honest Boundary

AVERA already has strong adapted sample workspaces.

What it does not yet have is a real externally sourced artifact example from a partner team.

This document exists to keep that gap explicit and actionable.

## What We Need Next

Best next real external artifact examples:

1. one `verification log CSV`
2. one `JUnit / xUnit XML`
3. one `simulation CSV`
4. one richer `requirements export`

## Why These Four

- verification log CSV: best first real pilot path
- JUnit / xUnit XML: easiest widely portable fallback
- simulation CSV: strong engineering value once a team is engaged
- requirements export: important supporting evidence layer

## What Counts As A Useful Example

A useful first external example should include:

1. one baseline artifact
2. one current artifact
3. one short change description
4. minimal requirement linkage if available
5. enough context to understand the review decision

## Collection Strategy

Do not ask for:

- historical archives
- multiple workflows
- full integrations

Ask only for:

- one representative export pair
- one repeated workflow
- one narrow review boundary

## What Happens After Collection

Once a real example is received:

1. map it to existing adapter path if possible
2. if not possible, define the minimum adapter extension
3. build one real pilot workspace
4. record the result in [AVERA_REAL_PILOT_EXECUTION_LOG_01.md](/Users/mac/Desktop/AVERA/docs/AVERA_REAL_PILOT_EXECUTION_LOG_01.md)
