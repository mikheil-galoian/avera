# AVERA Sample Export Request Template

**Date:** 2 May 2026  
**Status:** First sample-artifact request template  
**Purpose:** Ask a target team for the smallest useful export set needed to scope a narrow pilot

## Short Version

`To scope a narrow AVERA pilot, we do not need a broad integration. We only need one representative export set from one repeated review workflow.`

## Request Template

Use this message after a target team shows real interest:

`For the next step, we would like one small representative export set from the workflow you described. We are not asking for a platform integration or a large historical dataset. We only want enough material to test one narrow local-first review path.`

`The ideal package is:`

1. `one short change description`
2. `one baseline artifact export`
3. `one current artifact export`
4. `requirement linkage if available`
5. `component mapping or a workable substitute`

`If your workflow already has a repeated verification log export, that is the strongest first path for us. If not, JUnit/xUnit XML or simulation CSV can also work.`

## Best First Artifact Request

Preferred first artifact family:

- `verification log CSV`

Fallbacks:

- `JUnit / xUnit XML`
- `simulation CSV`

## Scope Discipline

Make this explicit:

- we do not need production integration
- we do not need all historical data
- we do not need multiple workflows at once
- we only want one repeated review path

## What We Are Evaluating

The purpose of the export is to test:

1. whether the artifact can be normalized cleanly
2. whether the review path is understandable
3. whether the resulting traceability and decision support are useful
4. whether the workflow is worth a narrow pilot

## What Counts As Enough

Enough for first scoping:

- one baseline example
- one current example
- enough context to understand what changed

Anything beyond that is optional for the first pass.
