# AVERA Adapted Evidence Runbook

**Date:** 1 May 2026  
**Status:** First narrow runbook for adapted-evidence pilots  
**Purpose:** Give one concrete operator path for running AVERA on workspaces built from adapted external artifacts

## What This Runbook Covers

This runbook is for AVERA scenarios where some core workspace inputs were
generated by adapters before analysis.

Current supported examples:

- `fixtures/adas-simulation-adapted`
- `fixtures/bms-requirements-adapted`
- `fixtures/bms-log-adapted`

## Operator Goal

Use the same AVERA kernel and review flow while making the external artifact
bridge explicit and inspectable.

## Preflight

Run:

```bash
cd /Users/mac/Desktop/AVERA
.venv/bin/python scripts/runtime_doctor.py
```

## Adapter Step

Choose the right adapter for the incoming artifact family.

### JUnit / xUnit

```bash
PYTHONPATH=src python3 -B -m avera adapt-junit \
  --input sample-results.xml \
  --out current_results.json \
  --run-id local-run \
  --stage hil
```

### Simulation CSV

```bash
PYTHONPATH=src python3 -B -m avera adapt-simulation \
  --input current-simulation-results.csv \
  --out current_results.json \
  --run-id adas-sim-current \
  --stage current
```

### Requirements Export Variant

```bash
PYTHONPATH=src python3 -B -m avera adapt-requirements \
  --input requirements-export.csv \
  --out requirements.csv
```

### Verification Log CSV

```bash
PYTHONPATH=src python3 -B -m avera adapt-logs \
  --input current-verification-log.csv \
  --out current_results.json \
  --run-id bms-log-current \
  --stage current
```

## Workspace Validation

After adapter output is in place:

```bash
PYTHONPATH=src python3 -B -m avera validate-workspace <workspace>
```

## Analysis Path

Run:

```bash
PYTHONPATH=src python3 -B -m avera analyze --project <workspace> --out <report_dir> --memory reports/avera-memory.jsonl
```

Then, when a full review bundle is needed:

```bash
PYTHONPATH=src python3 -B -m avera demo-refresh \
  --project <workspace> \
  --report-out <report_dir> \
  --memory reports/avera-memory.jsonl \
  --traceability-out reports/avera-traceability-index.json \
  --decision-out reports/avera-decision.json \
  --trend-out reports/avera-trend-index.json \
  --pack-out reports/avera-workspace-pack.json
```

## Shell Review Path

Launch:

```bash
./start_demo.sh
```

For adapted scenarios, the shell should show:

- scenario profile
- raw source artifacts
- normalized AVERA artifacts
- evidence-backed verdict
- adapted pilot mode and handoff readiness

## Success Criteria

This runbook is working when:

1. the adapted workspace validates
2. analysis produces the expected verdict path
3. the shell makes the adapted boundary visible
4. the handoff path remains the same as any standard AVERA workspace
