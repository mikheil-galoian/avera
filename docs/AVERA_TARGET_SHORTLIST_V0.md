# AVERA Target Shortlist v0

**Date:** 2 May 2026  
**Status:** First operating shortlist for outward motion  
**Purpose:** Narrow the first AVERA outreach cycle to a small number of high-fit target profiles

## Goal

This shortlist is not a company database.

It is a practical filter for the first outreach wave.

The aim is to avoid broad outreach and focus only on targets that are likely to:

- understand verification evidence pain
- already export usable artifacts
- accept a narrow local-first pilot

## Tier 1 Targets

Highest-priority target profiles:

1. automotive `Verification / Validation team`
2. ADAS validation group with repeated simulation or log review
3. BMS verification or release-readiness group with repeated regression triage

## Tier 1 Owner Roles

Best first owners:

1. `Validation Lead`
2. `Verification Lead`
3. `Test & Validation Manager`

## Tier 1 Artifact Families

Best-first artifact families:

1. `verification log CSV`
2. `JUnit / xUnit XML`
3. `simulation CSV`

## Tier 1 Use Cases

Best first use cases:

1. regression triage before release
2. baseline vs current verification comparison
3. release-review handoff with evidence
4. requirement-linked verification review

## Tier 2 Targets

Good next-step targets after first signal:

1. embedded ECU release-review teams
2. system integration teams with recurring evidence handoff friction
3. verification tooling owners who can sponsor a narrow internal pilot

## Deprioritized Targets

Do not prioritize first:

1. broad AI strategy contacts without workflow ownership
2. teams with no stable exportable artifacts
3. teams asking for hosted deployment first
4. groups wanting enterprise rollout before proving one workflow

## Success Condition

This shortlist is working if it helps identify:

1. one live target account
2. one owner role
3. one artifact family
4. one realistic first follow-up session
