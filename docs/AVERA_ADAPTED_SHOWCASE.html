Как это показывать

Короткий сценарий для первого design-partner разговора.

Порядок показа

  1. Сначала BMS case как знакомый change-review пример.
  2. Потом adapted ADAS simulation path как доказательство domain generality.
  3. Затем BMS requirements/log paths как доказательство intake flexibility.

Ключевая фраза

AVERA не просит идеальный новый workflow. Она берет уже существующие инженерные артефакты, нормализует их и строит объяснимый risk-review контур поверх них.

Команды запуска

  • DEMO_SCENARIO=adas-simulation-adapted ./start_demo.sh
  • DEMO_SCENARIO=bms-requirements-adapted ./start_demo.sh
  • DEMO_SCENARIO=bms-log-adapted ./start_demo.sh
Рекомендованный комплект для первого внешнего показа: этот showcase, adapted one-pager, adapted pilot script, adapted outreach packet и live shell там, где Streamlit runtime ведет себя стабильно. Если live shell не просыпается быстро, этот HTML остается чистым fallback asset без потери product message.
