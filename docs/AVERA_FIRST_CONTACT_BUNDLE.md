## Minimum Talk Track

Say:

`AVERA is a local evidence-first review layer for engineering changes. The current breakthrough is not a new dashboard. It is that external artifacts like simulation CSV, requirements exports, and verification logs can be normalized into one stable review path.`

Then say:

`We are not proposing a platform rollout. We are proposing one narrow pilot around one artifact family you already use.`

## Best First Pilot Shapes

- ADAS simulation regression review
- BMS requirements normalization before change review
- BMS verification-log normalization before release triage

## Good Next Step

If the conversation lands well, ask for:

1. one artifact family
2. one realistic sample export
3. one narrow review workflow
4. one follow-up session for pilot scoping

## What Not To Do

- do not pitch SaaS first
- do not lead with AI ambition language
- do not promise broad enterprise integration
- do not hide the raw-to-normalized boundary

## Current Product Message

AVERA does not require a perfect new process.  
It takes existing engineering evidence, normalizes it, and makes one release-review path easier to trust.
