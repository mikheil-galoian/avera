Process

From commit to verdict
in three steps.

1

Ingest evidence

Supply baseline and current test results, a requirements CSV, a component map, and a change description — via filesystem path, REST API call, or CI pipeline POST.

2

AVERA analyses

The engine evaluates threshold crossings, maps failures to requirements, scores confidence, triggers domain-specific safety rules, and derives a risk verdict.

3

Structured report

JSON report, Markdown narrative, evidence graph, traceability index, decision artifact, and workspace pack — ready for human review or automated gate enforcement.

Integration

Drop into any CI pipeline.

REST API with JSON-schema-validated request and response. X-API-Key authentication. Token rotation. Zero-downtime deployment.

# CI step — POST analysis inline, get structured verdict
curl -X POST https://api.avera.ai/analyze/inline \
  -H "X-API-Key: $AVERA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '@avera-payload.json'

# Response
{
  "verdict": "confirmed_regression",
  "risk": "release_blocking",
  "confidence_score": 0.95,
  "affected_components": ["BMS Thermal Control"],
  "schema_version": "avera.assessment.v0.2"
}
